O GitHub Actions nos ajuda a automatizar algumas etapas do nosso ciclo de vida do desenvolvimento de software.
Ele é dirigido a eventos, ou seja, é possível pedir que o Actions realize algumas tarefas após um evento específico ser identificado.
Para uma lista completa dos eventos disponíveis e sua sintaxe, acesse.
No entanto a sintaxe básica estará exposta na lista a baixo.

Entendendo um arquivo yml básico do actions. As aspas triplas('''Livre''') serão o identificador de quando o código ali dentro é escolha do programador

name: ____'''Opcional, é o nome do fluxo de trabalho que vai aparecer na aba Actions do GitHub.'''
on: ______['''Especificar o/s evento/s que engatilha/m o fluxo de trabalho.''']
jobs: ____{Aqui não vai código, é um agrupador de todos os jobs que serão realizados}
____'''Nome do Job''':
____ ___runs-on: ____'''Escolha entre as opções {Serve para configurar o sistema operacional da máquina virtual que vai rodar seu fluxo de trabalho.}
____ ___steps: {Agrupador de actions ou linhas de comando shell que formam o Job.}
____ ___ ___-uses: ____'''Por exemplo: actions/checkout@v2''' {Informa ao Job que ele deve procurar uma action já existente na comunidade}
____ ___ ___-run: ____'''Por exemplo: python -m pip install --upgrade pip''' {informa ao Job que ele deve executar uma linha de comando no shell.}

    Sintaxe YAML:

  • Eventos que engatilham fluxos de trabalho:
    •

    1. Eventos únicos:
      2.  on: push

    2. Eventos em listas:
      3.  on:[push, pull_request]

    3. Eventos múltiplos e branches:
      (ignore os underlines)
      4.  on:
      ____push:
      ________branches:
      ____________- main
      ____pull_request:
      ________branches:
      ____________- main
