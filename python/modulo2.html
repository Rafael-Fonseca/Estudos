  • PyUp:
    •  Serviço de automatização que ajuda a manter as bibliotecas(dependências) do projeto atualizadas.
    Necessário ter conta no PyUp, vinculada ao seu repositório Git.
    Dar permissão para que o PyUp leia e escreva no seu repositório.
    Na raiz do projeto crie e configure o arquivo .pyup.yml

  • PyPi(pip):
    •  Empacotador de bibliotecas que possibilita reaproveitar código de forma correta e compartilhada com a comunidade python.
    Para realizar a distribuição, é necessário:

    Criar e configurar o arquivo setup.py na raiz do projeto.
    Executar o comando git tag {nome da tag, normalmente coincide com o número da versão} Este comando cria uma tag.
    Executar o comando git push --tags, este comando envia as tags criadas e vincula ao último comit.
    Executar python setup.py sdist, este comando cria uma distribuição .tar.gz
    Instale se necessário a biblioteca twine
    Execute o comando twine upload dist/* {Este comando envia suas releases para o pip.

  • Pytest:
    •  Biblioteca que possibilita criar testes.

  • Coverage:
    •  É um conceito onde que mostra a porcentagem de código que está coberta por testes.
    A biblioteca utilizada para isso é uma extensão da pytest e se chama pytest-cov. Também foi utilizado a biblioteca coverage para gerar o relatório de cobertura.

  • Test Driven Development:
    •  Desenvolvimento dirigido a testes(TDD) é uma forma de desenvolver código, onde os testes são realizados antes
    dos códigos. O desenvolvimento ocorre em passos, onde cada passo é dado de forma a passar no próximo teste.

  • Baby Steps:
    •  Completa o TDD, fazendo com que cada passo seja realizado da maneira mais simples possível, tanto para a criação
    do teste, quanto para a sua solução.

  • Pyenv:
    •  Serve para você poder ter mais de uma versão python instalada, é necessário realizar a instalação e editar o arquivo .bashrc de acordo com o comando pyenv init.
    A vantagem de ter mais de uma versão instalada é a possibilidade de isolar o seu sistema de desenvolvimento, evitando danos ao seu sistema operacional.

  • Pipenv:
    •  Use o pip para instalar o pipenv e configure o .bashrc para que o ambiente virtual gerado por ele fique na mesma pasta do seu projeto.
    pipenv install -d é o comando que lê os arquivos Pipfile e Pipfile.lock, cria um ambiente virtual e instala as dependências, incluindo as de desenvolvimento.
