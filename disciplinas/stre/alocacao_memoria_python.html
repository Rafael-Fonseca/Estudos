Como funciona a alocação de memória em Python?


Em python este processo é realizado pelo Python Memory Manager.
Python é implementado utilizando a linguagem C e este processo ocorre no modo C.
Python possui um coletor de lixo embutido, ele utiliza a técnica contagem de referência, nela, um objeto é desalocado da memória quando o programa não possui mais referências a ele.
Desta forma, o desenvolvedor python não precisa se preocupar em criar um coletor de lixo.
Preocupação esta que deve ter um desenvolvedor da linguagem C, que nativamente não implementa um coletor de lixo.

A alocação da memória pode ocorrer de duas formas:

  1. (Para memória estática) -- Pilha:
    2. Armazena a memória necessária apenas em uma determinada função ou chamada de método.
    Quando uma função é chamada, ela é adicionada à pilha de chamadas do programa.
    Quaisquer atribuições de memória local - como inicializações de variáveis - dentro destas funções são armazenadas temporariamente na pilha de chamadas de função,
    e são excluídas assim que a função retorna. Então a pilha de chamadas segue para a próxima tarefa.
  2. (Para memória estática) -- heap:
    3. A alocação de heap armazena a memória usada no programa, no escopo global. Essas variáveis são necessárias fora do método específico ou são compartilhadas em várias funções globalmente.

Práticas recomendadas em python para alocação de memória ou complexidade de espaço.

  1. Evitar fatiamento de listas:
    2. Tentar usar parâmetros de função ou variáveis separadas para rastrear índices ao invéz de fatiar ou alterar uma lista. O fatiamento gera uma nova lista.
  2. Preferir "for each" ao invés de "for in range":
    3. Se o índice não for necessário e seu programa funcionar sem ele, não o aloque!
